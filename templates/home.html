CommerceHub API
View on GitHub Open API Docs

A production-ready E-commerce REST API built with Django REST Framework, PostgreSQL, JWT authentication, and Stripe integration.

Supports authentication, product management, cart operations, order processing, inventory tracking, Stripe payments, filtering, pagination, throttling, and Swagger API documentation.

Interactive Swagger/OpenAPI documentation available at:

/api/docs/

Backend Highlights

  • Clean RESTful API design
  • JWT authentication with protected endpoints
  • UUID-based primary keys across products, orders, categories, and cart resources
  • Stripe payment processing
  • Custom object-level permissions
  • Nested routing
  • DRF filtering, searching, ordering, and pagination
  • Throttling (rate limiting)
  • Production-aware architecture decisions

Tech Stack

  • Python
  • Django
  • Django REST Framework
  • PostgreSQL
  • JWT authentication
  • Stripe API
  • Docker
  • drf-spectacular (Swagger documentation)

Purpose & User Flow

Users can:

  • Register and authenticate using JWT
  • Browse and search products
  • Manage their cart (add/remove items)
  • Manage cart item quantities
  • Checkout and place orders
  • Make payments via Stripe
  • Track order status
  • View order history

Admin users can:

  • Manage products and categories
  • View and manage all orders
  • Update order statuses
  • Track inventory levels

How to Test

  1. Register a user (/api/auth/register/)
  2. Login and obtain a JWT token (/api/auth/login/)
  3. Authorize using the access token in Swagger
  4. Create product categories (admin only)
  5. Create products linked to categories (admin only)
  6. Browse products with filtering, searching, and ordering
  7. Add products to cart
  8. Update cart item quantities
  9. Checkout and create an order
  10. Create Stripe payment intent
  11. View user orders and order details
  12. Update order status as admin
